输入

·

在Java中，常见的输入方式有以下几种，它们的主要区别在于使用场景和功能：

1. Scanner 类

  • 用途: 用于从控制台、文件或其他输入流中读取数据。

    支持多种数据类型（如 int, double, String 等）。

    需要处理输入异常（如 InputMismatchException）。

  • 示例:

2. BufferedReader 类

  • 用途: 用于高效读取字符输入流，通常与 InputStreamReader 结合使用。

    读取效率高，适合处理大量数据。

    只能读取**字符串**，需手动转换数据类型。

    需要处理 IOException。

  • 示例:

3. Console 类

  • 用途: 用于从控制台读取输入，支持密码输入（输入时不显示）。

  • 特点:

    • 仅适用于控制台输入。
    • 支持密码输入（readPassword 方法）。
    • 在IDE中可能无法使用。

  • 示例:

4. System.in 与 InputStream

  • 用途: 直接使用 System.in 读取字节流。

  • 特点:

    • 读取的是字节数据，需手动转换为字符或其他类型。
    • 灵活性高，但使用复杂。

  • 示例:

5. DataInputStream 类

  • 用途: 用于从输入流中读取原始数据类型（如 int, double 等）。
  • 特点:
    • 适合读取二进制数据。
    • 需与 FileInputStream 等结合使用。
  • 示例:

总结

  • Scanner: 简单易用，适合控制台输入。
  • BufferedReader: 高效读取字符串，适合大量数据。
  • Console: 适合控制台输入，支持密码输入。
  • System.in: 直接读取字节流，灵活性高但复杂。
  • DataInputStream: 适合读取二进制数据。

根据具体需求选择合适的输入方式。

输出

在 Java 中，输出方式也有多种选择，具体使用哪种方式取决于需求和场景。以下是常见的输出方式及其区别：

1. System.out.print() 和 System.out.println()

  • 用途: 最常用的输出方式，适合大多数情况。

    System.out.print(): 输出内容，不换行。

    System.out.println(): 输出内容并换行。

    简单易用，适合调试和输出少量数据。

  • 缺点:

    • 性能较低，不适合输出大量数据（如 10^5 以上的数据量）。

  • 示例:

2. StringBuilder 或 StringBuffer

  • 用途: 适合需要频繁拼接字符串的场景，尤其是输出大量数据时。

    StringBuilder: 非线程安全，性能高。

    StringBuffer: 线程安全，性能稍低。

    通过 append() 方法拼接字符串，最后一次性输出。

  • 优点:

    • 性能高，适合输出大量数据。

  • 缺点:

    • 需要手动管理字符串拼接。

  • 示例:

3. PrintWriter

  • 用途: 适合需要高效输出大量数据的场景（如算法竞赛）。

    可以输出到控制台或文件。

    性能高，适合输出大规模数据。

    支持格式化输出（类似 System.out.printf）。

  • 优点:

    • 性能优于 System.out。
    • 可以输出到文件，适合需要保存结果的场景。

  • 缺点:

    • 使用稍复杂，需要手动刷新和关闭。

  • 示例:

4. System.out.printf()

  • 用途: 适合需要格式化输出的场景。

  • 特点:

    支持类似 C 语言的格式化输出。

    语法：System.out.printf("格式字符串", 参数1, 参数2, ...)。

  • 优点:

    • 格式化输出方便。

  • 缺点:

    • 性能较低，不适合输出大量数据。

  • 示例:

5. BufferedWriter

  • 用途: 适合需要高效输出大量数据的场景（如算法竞赛）。

  • 特点:

    • 性能高，适合输出大规模数据。
    • 通常与 FileWriter 或 OutputStreamWriter 结合使用。

  • 优点:

    • 性能优于 System.out。
    • 可以输出到文件。

  • 缺点:

    • 使用稍复杂，需要手动刷新和关闭。

  • 示例:

6. Files.write()（Java 7+）

  • 用途: 适合将数据直接写入文件。
  • 特点:
    • 简单易用，适合一次性写入文件。
    • 支持写入字符串列表或字节数据。
  • 优点:
    • 代码简洁。
  • 缺点:
    • 不适合需要频繁写入的场景。
  • 示例:

总结

  • 简单输出: 使用 System.out.print() 或 System.out.println()。
  • 格式化输出: 使用 System.out.printf()。
  • 高效输出大量数据: 使用 PrintWriter 或 BufferedWriter。
  • 文件输出: 使用 Files.write() 或 BufferedWriter。

在算法竞赛中，**PrintWriter 或 BufferedWriter** 是最常用的输出方式，因为它们性能高且适合大规模数据输出。而日常练习或调试时，System.out.println() 是最方便的选择。

1. 面向对象编程（OOP）核心概念

  • 类（Class）：类是对象的模板，用于定义对象的属性和行为。
  • 对象（Object）：对象是类的实例，具有状态（属性）和行为（方法）。
  • 封装（Encapsulation）：将数据和行为封装在类中，隐藏内部实现细节。
  • 继承（Inheritance）：一个类继承另一个类的属性和方法，实现代码复用。
  • 多态（Polymorphism）：一个接口或父类引用指向子类对象，实现动态绑定。
  • 抽象（Abstraction）：通过抽象类和接口定义通用的行为规范。

3. 关键知识点

3.1 抽象类 vs 接口

  • 抽象类：
    • 可以包含抽象方法和非抽象方法。
    • 子类通过 extends 继承。
    • 用于定义通用的行为模板。
  • 接口：
    • 只能包含抽象方法（Java 8 后可以包含默认方法）。
    • 类通过 implements 实现。
    • 用于定义行为契约。

3.2 方法重写（Override）

  • 作用：子类重新定义父类的方法。
  • 规则：
    • 方法名、参数列表和返回类型必须一致。
    • 访问权限不能比父类方法更严格。
    • 使用 @Override 注解明确表示重写。

3.3 多态

  • 作用：一个接口或父类引用指向子类对象。
  • 实现方式：通过方法重写和接口实现。
  • 优点：提高代码的灵活性和可扩展性。

5. 总结

  • 通过抽象类和继承，可以实现代码的复用和扩展。
  • 多态特性使得程序更加灵活，易于维护。
  • 使用 @Override 注解可以提高代码的可读性和安全性。
